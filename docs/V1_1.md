## Objetivos da 1.1

1. **Care Chart** — prontuário clínico estendido além do PEP textual
2. **Medicação** — prescrições ativas/suspensas com posologia
3. **Exames** — pedidos e resultados vinculados ao paciente
4. **Protocolos** — templates reutilizáveis e matrículas por beneficiário

## Entregas v1.1.0 (produção)

### Schema

| Entidade | Função |
|----------|--------|
| `PatientClinicalProfile` | Alergias, comorbidades, tipo sanguíneo, observações |
| `MedicationPrescription` | Prescrições com status e posologia |
| `ExamOrder` | Pedidos de exame com status e resultado |
| `CareProtocolTemplate` | Templates de protocolo (interno) |
| `PatientProtocolEnrollment` | Matrícula do beneficiário em protocolo |

### Portais

| Portal | Entrega |
|--------|---------|
| **Prestador** | `/prestador/pacientes`, Care Chart no atendimento e histórico |
| **Interno** | Aba clínica no Cliente 360°, painel de templates em cadastros |
| **Beneficiário** | Resumo clínico (meds, exames, protocolos) |

### Seed e testes

- `prisma/seed-data/clinical-demo.ts` — massa Care Chart (João Pereira: perfil HAS/DM2, Losartana, HbA1c, protocolo HAS)
- Demo: login `joao.pereira@email.com` / `bibi123` → portal beneficiário ou atendimento via prestador
- Testes automatizados dedicados ao módulo clínico: **pendentes** (suíte Vitest cobre PPU, RBAC e auth)

### Serviços (`src/lib/`)

| Serviço | Função |
|---------|--------|
| `clinical-profile-service.ts` | CRUD `PatientClinicalProfile` (alergias, comorbidades, tipo sanguíneo) |
| `medication-service.ts` | Prescrições e transições de status (`ATIVA` → `SUSPENSA` → `ENCERRADA`) |
| `exam-order-service.ts` | Pedidos de exame e laudo (`SOLICITADO` → … → `LAUDADO`) |
| `care-protocol-service.ts` | Templates e matrículas com checklist JSON |
| `clinical-overview.ts` | Visão consolidada (perfil + meds ativas + exames pendentes + protocolos) |
| `clinical/constants.ts` | Status, labels e parsers JSON |

### APIs REST

**Prestador** (`role: PRESTADOR`)

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| `GET` | `/api/prestador/patients` | Lista pacientes com atendimentos do provider (`?q=` busca) |
| `GET` | `/api/prestador/patients/{id}/clinical-overview` | Visão consolidada Care Chart |
| `GET\|PUT` | `/api/prestador/patients/{id}/clinical-profile` | Perfil clínico (alergias, comorbidades) |
| `GET\|POST` | `/api/prestador/patients/{id}/medications` | Listar / prescrever medicação |
| `PATCH` | `/api/prestador/medications/{id}` | Alterar status da prescrição |
| `GET\|POST` | `/api/prestador/patients/{id}/exam-orders` | Listar / solicitar exame |
| `PATCH` | `/api/prestador/exam-orders/{id}` | Status, agendamento, resultado |
| `GET\|POST` | `/api/prestador/patients/{id}/protocols` | Matrículas + templates disponíveis |
| `PATCH` | `/api/prestador/protocols/{id}` | Checklist e status da matrícula |

**Interno** (`requireInternoModule("cadastros")`)

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| `GET` | `/api/interno/patients/{id}/clinical` | Visão clínica no Cliente 360° |
| `GET\|POST` | `/api/interno/protocol-templates` | Templates de protocolo |
| `PATCH\|DELETE` | `/api/interno/protocol-templates/{id}` | Editar / desativar template |

**Beneficiário** (`role: BENEFICIARIO`, escopo `user.patientId`)

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| `GET` | `/api/beneficiario/clinical` | Resumo clínico (somente leitura) |

### UI

| Componente | Onde |
|------------|------|
| `ClinicalCarePanel` | Atendimento (`AtendimentoView`) e histórico (`PrestadorPatientHistoryView`) |
| `ClinicalSidebar` | Sidebar com resumo (alergias, meds, exames pendentes) |
| `ProtocolTemplatesPanel` | Cadastros interno → aba **Protocolos clínicos** |
| Seções clínicas | `PatientOverviewView` (Cliente 360°), `BeneficiarioView` (meds, exames, protocolos) |
