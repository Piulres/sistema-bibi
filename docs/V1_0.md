## Escopo base (v1.0.0)

POC SaaS HealthTech multi-tenant com quatro portais segregados por `role`:

| Portal | Rota | Papel |
|--------|------|-------|
| Prestador | `/prestador` | Agenda e atendimento clínico |
| Interno | `/interno` | Operação, faturamento Pay Per Use, CRM |
| Empresa PJ | `/pj` | Visão corporativa e consumo |
| Beneficiário | `/beneficiario` | Self-service, agenda, prontuário |

### Núcleo de negócio

- Faturamento **Pay Per Use** (procedimentos efetivamente usados)
- Precificação dinâmica por empresa (`PricingRule`)
- PIX mock, assinaturas recorrentes, webhooks B2B
- Cliente 360° (`getPatientOverview`) no interno e beneficiário
- RBAC interno (`ADMIN`, `FATURAMENTO`, `RECEPCAO`, `READONLY`)
- MFA TOTP, telemedicina mock, TISS XML, white-label
- Walk-in particular, mapas CRUD/fluxo, melhorias visuais (PRs #64–#66)

Mapa completo: **27 entidades** em `src/lib/crud-operations-map.ts`.

## Incrementos v1.0.1 (na `main` — PRs #69–#71, release #73)

### Agenda prestador completa (PR #69)

- API `GET /api/prestador/agenda?view=day|upcoming|past&date=`
- Abas **Dia / Próximos / Histórico** em `AgendaView`
- Histórico do paciente: `/prestador/paciente/[id]` + `GET /api/prestador/patients/[id]/overview`
- Link “Histórico completo” na tela de atendimento

### Modos demo e operação (PR #71)

- Dual SQLite: `demo.db` (massa seed) + `operation.db` (bootstrap mínimo, dados reais)
- Seletor em **`/interno/seguranca`** → card **Base de dados — demo ou operação** (ADMIN)
- Persistência operação via Netlify Blobs (`sqlite-blob-persistence.ts`)
- Variáveis: `DUAL_DATA_STORE`, `DATA_STORE_MODE`, `APP_MODE`, `RUN_SEED_ON_BUILD`
- Docs: [`OPERACAO_DADOS.md`](OPERACAO_DADOS.md)

### Processo e qualidade

- Fluxo **dev-first**: PRs integram em `dev` antes de `main` (PR #70)
- Build CI sem `DATABASE_URL` no prerender dos portais
- Teste PPU com slot determinístico
- Refator `getPrisma()` centralizado em serviços
- **101** testes unit/API · **44+** E2E
