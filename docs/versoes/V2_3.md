## 1. Resumo executivo

A v2.3 evolui o **onboarding guiado** para um modelo em **duas fases** (tour principal + micro-tours por módulo), adiciona o segmento **Engenharia Civil** (ERP empreiteira) e consolida a **documentação OpenAPI** com Swagger UI.

---

## 2. Onboarding v3 ([#152](https://github.com/Piulres/sistema-bibi/pull/152))

Detalhe completo: [`produto/ONBOARDING_TOUR.md`](../produto/ONBOARDING_TOUR.md)

| Antes (v2.2) | Agora (v2.3) |
|--------------|--------------|
| Tour linear longo (17 passos no interno) | Tour principal ~5 passos + micro-tours por rota |
| Passos de página filtrados — raramente vistos | Micro-tour na **1ª visita** a cada módulo |
| Pular não persistia | `dismissed` no `localStorage` |
| Nav invisível no mobile | `data-tour-nav` no drawer + botão Tour visível |
| `ONBOARDING_VERSION = 2` | `ONBOARDING_VERSION = 3` |

**Hotspots comerciais:** Cliente 360°, PIX beneficiário, PEP prestador, demo reset, dual-store, mapa CRUD.

**Código:** `src/lib/onboarding/` · `src/components/onboarding/`

**Testes:** `tests/unit/onboarding.test.ts` (21 casos)
