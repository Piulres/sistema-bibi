## Benchmark de mercado

Funcionalidades alinhadas a sistemas como iClinic, Feegow, CPC Brasil e Onihosp:

| Capacidade | Implementação Bibi |
|------------|-------------------|
| Catálogo por categoria (medicamento, material, OPME, insumo) | `MedicalProduct` + portal Interno |
| Controle por lote e validade (RDC 304/2019) | `StockLot` + alertas automáticos |
| Entrada / saída / ajuste / perda / devolução | `StockMovement` + FIFO |
| Dispensação por paciente | Atendimento prestador + movimentação `DISPENSACAO` |
| Kit de materiais por procedimento | `ProcedureMaterialKit` + baixa no Pay Per Use |
| Alertas operacionais | Estoque mínimo, vencimento (90/30 dias), lotes bloqueados |
| Auditoria | Timeline universal (`STOCK_*` actions) |

## Entregas v1.3.0

### Portal Interno — `/interno/estoque`

- Resumo executivo (produtos, lotes, valor em estoque, alertas)
- CRUD de produtos médicos
- Entrada de lotes com custo e fornecedor
- Movimentações manuais (saída, perda, ajuste, devolução)
- Kits de materiais vinculados a procedimentos
- RBAC: `ADMIN` e `RECEPCAO`

### Portal Prestador — atendimento

- Nova aba **Materiais** no fluxo clínico
- Dispensação manual vinculada ao agendamento/paciente
- Baixa automática do kit ao registrar procedimento (Pay Per Use)

### APIs

| Rota | Método | Descrição |
|------|--------|-----------|
| `/api/interno/stock/products` | GET, POST | Catálogo + overview |
| `/api/interno/stock/products/[id]` | PATCH | Atualizar produto |
| `/api/interno/stock/lots` | GET, POST | Lotes / entrada |
| `/api/interno/stock/lots/[id]` | PATCH | Status do lote |
| `/api/interno/stock/movements` | GET, POST | Histórico / movimentação |
| `/api/interno/stock/alerts` | GET | Alertas operacionais |
| `/api/interno/stock/procedure-kits/[procedureId]` | GET, POST | Kit por procedimento |
| `/api/prestador/appointments/[id]/materials` | GET, POST | Dispensação no atendimento |

### Seed demo
