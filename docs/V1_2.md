## Objetivos da 1.2

Consolidar em um único pacote de produção:

1. **Care Chart** (herdado da 1.1) — prontuário estendido, medicação, exames, protocolos
2. **Cadastros de mercado** — validação CPF/CNPJ, campos B2B, precificação por empresa
3. **Portais** — a11y, navegação mobile, dashboard prestador, rotas dedicadas beneficiário
4. **Auditoria** — timeline universal, precificação B2B auditável, edição de assinaturas
5. **Exportações** — PDF, Excel e CSV nos quatro portais
6. **Homepage** — landing moderna com SEO e JSON-LD

### #85 — Exportações

Infraestrutura compartilhada em `src/lib/exports/`:

| Camada | Arquivos |
|--------|----------|
| Formatos | `format.ts` — `pdf`, `xlsx`, `csv`, `json` |
| Tabular | `tabular.ts` (ExcelJS + PDFKit) |
| Domínio | `builders.ts`, `pep-pdf.ts`, `invoice-pdf.ts` |
| UI | `ExportButtons.tsx` — links `?format=` |

**Rotas principais:**

| Portal | Rota | Formatos |
|--------|------|----------|
| Interno faturamento | `GET /api/interno/billing/export` | xlsx (default), pdf, csv |
| Interno fatura | `GET /api/interno/invoices/[id]/export` | pdf (default) |
| Interno relatórios | `GET /api/interno/reports?type=billing\|crm` | csv (default) |
| Interno auditoria | `GET /api/interno/audit/export` | xlsx (default) |
| Interno LGPD | `GET /api/interno/patients/[id]/export` | json sem `section`; tabular com `section` |
| Prestador extrato | `GET /api/prestador/extrato/export` | xlsx |
| Prestador PEP | `GET /api/prestador/records/[recordId]/export` | **somente pdf** |
| PJ | `GET /api/pj/reports` | csv (default) |
| Beneficiário | `GET /api/beneficiario/export` | xlsx — `section` por área |

**UI:** botões `ExportButtons` nas views de faturamento, relatórios, Cliente 360°, auditoria, extrato prestador, PJ e beneficiário.

Detalhe de fluxos: [`FLUXOS.md`](FLUXOS.md) §8.8 · Arquitetura: [`ARQUITETURA.md`](ARQUITETURA.md) §21
